## 3. `joints` 与轨迹几何关系：YZ / XZ / XY 哪个平面最接近？

我用下面这 8 个三点夹角定义，从轨迹表直接重建了关节角：

- `Hip = angle(RPEL, RHIP, RKNE)`
- `Kne = angle(RHIP, RKNE, RANK)`
- `Ank = angle(RKNE, RANK, RMTP)`
- `Mtp = angle(RANK, RMTP, RHTOE)`
- `Sho = angle(RSCA, RSHO, RELB)`
- `Elb = angle(RSHO, RELB, RWRI)`
- `Wri = angle(RELB, RWRI, RMCP)`
- `Mcp = angle(RWRI, RMCP, RFTOE)`

- `XZ` 和 `XY` 都远差于 `YZ`，说明这个文件里的 `joints` 基本上就是按与 `YZ` 接近的平面定义出来的

## 4. 一个重要异常：`RHIP` 掉点会在 `joints` 里产生突变

- `RHIP` 从帧 `61099` 开始缺失，一直到文件结束
- 在同一时刻，`Hip` 和 `Kne` 在 `joints` 表中发生了非常大的跳变：

| joint   |   before |   after |   abs_jump_deg |
|:--------|---------:|--------:|---------------:|
| Hip     |   93.895 |  18.935 |         74.96  |
| Kne     |   84.67  | 137.963 |         53.293 |

- 这说明 `joints` 不是完全独立的真值，更像是由 marker 轨迹计算出来、并且对原始 marker 质量敏感的派生标签
- 对建模来说，最后约 `4.1 s` 的 `Hip/Kne` 需要特别小心，最好加质检或屏蔽

## 5. 对建模最有用的结论

1. `joints` 比原始 `XYZ` 更适合作为第一条主线目标：维度更低、平移干扰更少、解释更直接。
2. 这个文件里，`joints` 与 `YZ` 投影几乎是一一对应的几何关系；所以如果要做“简化版 marker 目标”，优先考虑 `YZ`，不是 `XZ`。
3. 但 `joints` 不是完美真值。`RHIP` 掉点会污染 `Hip/Kne`，说明建模前仍要做 session 级 QC。
4. `Mcp` 明显比其他 7 个角度更不一致，可能是角度定义、远端点选择或手/蹄端几何定义不同，建议单独核查。
