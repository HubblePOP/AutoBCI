# 步态脑电二分类：数据切分与测试方案

> 更新日期：2026-04-14

***

## 一、我们要做什么

用脑电信号（ECoG）判断猪在走路时，当前处于**支撑期**还是**摆动期**。

这是一个**二分类任务**，不是连续轨迹预测。

```
支撑期（stance）：脚接触地面的阶段
摆动期（swing）：脚离开地面往前迈的阶段

一个完整步态周期：
  ... ─┤支撑├─┤摆动├─┤支撑├─┤摆动├─ ...
       0      1      0      1
```

为什么不直接预测连续轨迹？因为已有实验表明连续预测存在明显瓶颈：

**算法老师之前的连续轨迹预测结果（脑电 → 关节角度）：**

| 算法                  | 平均皮尔森 r | 平均 RMSE | 备注               |
| ------------------- | ------- | ------- | ---------------- |
| DeepConvNet（当前最优）   | 0.675   | 132.9   | 后肢最好 RKne y=0.74 |
| Sinc-ShallowNet（基准） | 0.393   | 40.0    | 整体偏弱             |

关键发现：

- **z 轴（高度方向）大面积 nan**——DeepConvNet 在多数关节的 z 轴完全无法预测
- y 轴（行进方向）相对较好，r 可到 0.70-0.76
- 说明脑电对"关节在哪个高度"的信息很弱，但对"关节在行进方向上的位置"有一定信号

对比：**纯运动学预测**连续轨迹，跨试次 r = 0.97（说明运动本身非常有规律）

所以脑电可能不包含精细的连续坐标信息（尤其是 z 轴），但可能包含"当前在哪个运动阶段"的粗粒度信号。我们先降维：不问"关节在哪"，只问"是支撑还是摆动"。

## 二、参考标签怎么来的

参考标签来自运动捕捉数据，不来自脑电。

具体是用右后脚趾（`RHTOE_z`）和右前脚趾（`RFTOE_z`）的 z 轴数据，通过规则方法切分出支撑期和摆动期。

我们试了 4 种规则方法：

| 方法    | 试次可用率 | 说明                     |
| ----- | ----- | ---------------------- |
| 极值包络法 | 100%  | z 轴局部极小值 = 落地，极大值 = 抬脚 |
| 微分过零法 | 100%  | z 轴速度过零点切分             |
| 滞回阈值法 | 91%   | 双阈值切换，减少抖动             |
| 双脚联合法 | 91%   | 前后脚联合判断                |

当前暂定使用**极值包络法**作为参考标签（provisional v1），因为它在 22 条 clean64 试次上全部可用。

**注意**：这些标签是暂定的，边界精度还在验证中。但对于"能不能分出支撑/摆动"这个二分类任务来说，已经够用了。

***

## 三、核心难点

### 问题

与运动想象范式不同，步态运动是**连续发生**的：

- **没有外部提示音**——没有人告诉算法"现在开始分类"
- **边界不确定**——支撑和摆动之间的过渡是渐变的，不是瞬间切换
- **存在固有延迟**——神经信号到肌肉执行之间有传导延迟，脑电窗口和运动时刻之间的最佳对齐点是未知的
- **完美预测也不可能完全同步**——即使把运动学答案原样输入，由于延迟也不会和真实运动完全对齐

这意味着两件事需要先搞清楚：

1. **在哪里取样**——边界附近的样本标签本身就是模糊的，应该避开
2. **脑电窗口应该对齐到运动时刻的什么位置**——提前多少、取多长，直接决定算法能不能看到有效信息

## 四、参考思路：取稳定区间锚点 + 扫描时延

以下是我们目前的一种参考思路，供参考和讨论。

### 思路：回避边界，只在稳定区间内部取样

不在边界附近取样，而是取**每个阶段的稳定区间内部**。

```
一个步态周期示意：

z值
 ^
 |      /\        /\
 |     /  \      /  \
 |    /    \    /    \
 |___/      \__/      \___
 |
 +--[===]----[===]----[===]--→ 时间
     支撑     摆动     支撑
      ↑        ↑
    取这里    取这里
   （稳定区）（稳定区）

 不取边界 ──→  ↕ ↕  ←── 不取边界
```

具体做法：

1. 用运动学标签找到每段支撑期和摆动期
2. 找到每段的**中点**（或者说"最稳定的那一刻"）
3. 从中点往前取一段脑电窗口
4. 这段脑电就是输入，对应的阶段标签（0 或 1）就是输出

这样做的好处是：标签明确（在稳定区的中心，不存在"到底算支撑还是摆动"的歧义），而且如果连稳定区间内部都分不对，边界附近更不可能。

### 关键参数：窗长和时延

```
时间轴 ──────────────────────────────→

           ┌── 脑电窗口 ──┐
           │               │
    ←─lag──┤               ├── 锚点（阶段中点）
           │               │
           └───────────────┘
           ← window_seconds →
```

- **window\_seconds**：脑电窗口长度。越长信息越多，但也越容易跨越阶段边界
- **global\_lag\_ms**：锚点往前偏移多少毫秒。因为神经信号比运动提前，所以脑电窗口应该在运动发生之前

### 建议扫描的参数组合

**窗长（window\_seconds）：**

| 窗长   | 含义           |
| ---- | ------------ |
| 0.1s | 极短窗，只看瞬时特征   |
| 0.5s | 约半个步态周期      |
| 1.0s | 约一个完整步态周期    |
| 3.0s | 多个步态周期，看长程节律 |

**时延（global\_lag\_ms）——锚点往前偏移：**

| 时延     | 含义                |
| ------ | ----------------- |
| 0ms    | 脑电窗口结束于锚点本身       |
| 100ms  | 提前 100ms，补偿基本神经延迟 |
| 300ms  | 提前 300ms          |
| 500ms  | 提前半秒              |
| 1000ms | 提前 1 秒            |
| 3000ms | 提前 3 秒，看是否有远程预测信号 |

**建议的扫描策略：**

1. 先固定窗长 = 1.0s，扫所有时延 → 找到最佳时延
2. 再固定最佳时延，扫所有窗长 → 找到最佳窗长
3. 最后在最佳组合附近做细扫

如果最好的结果也只有 50-55%（接近随机猜），说明这组参数下脑电确实不包含步态阶段信息。

当然，这只是一种参考方案。如果有更好的取样和对齐策略，欢迎直接尝试。

## 四、评分方式

### 主指标：平衡准确率（Balanced Accuracy）

因为支撑期和摆动期的时长不等（支撑期约占 60%），所以不用普通准确率，用平衡准确率：

```
balanced_accuracy = (支撑期正确率 + 摆动期正确率) / 2
```

- 50% = 随机猜
- 60% = 略好于随机
- 70%+ = 有意义的信号
- 80%+ = 强信号

### 补充指标

| 指标     | 说明            |
| ------ | ------------- |
| 试次可用率  | 能产出有效预测的试次占比  |
| 各类别 F1 | 支撑期和摆动期分别的 F1 |
| 混淆矩阵   | 看是不是某一类特别差    |

## 五、补充说明

### 为什么要扫时延

脑电信号和运动之间存在时间差：

- 运动皮层的计划信号可能**领先于**实际运动几百毫秒
- 但信号采集和处理也会引入额外延迟

我们不知道最佳时间对齐点在哪里。所以需要系统地扫描，找到那个"脑电最能区分支撑/摆动"的时间窗口。

如果所有时延下分类都是随机水平，说明脑电里可能确实不包含步态阶段信息——至少在当前的采样和信号质量下。

### 如果单次分类都做不到呢

这就是我们设计这个测试的核心逻辑：

```
单次锚点二分类（控制变量，最简单）
         ↓ 如果做不到
逐帧连续分类（更难，需要算法择时）
         ↓ 如果做不到
连续轨迹预测（最难）
```

如果最简单的、控制变量的、只在稳定区间���锚点的二分类都做不到，那后面的连续预测更不可能。这就是我们需要先确认的事情。

## 六、数据格式

### 输入

每个样本是一段脑电窗口：

- 形状：`(n_channels, n_timepoints)`
- 采样率：与原始脑电一致
- 通道：全部 ECoG 通道
- 预处理：CAR + 陷波 + 带通（具体参数见数据配置文件）

### 输出

- 标签：`0`（支撑）或 `1`（摆动）

### 数据划分

- 按试次（trial）划分，不按时间点
- train / val / test 划分与现有配置一致
- **不允许**同一条试次同时出现在训练集和测试集

## 七、下一步

1. 按上面的参数组合做系统扫描
2. 重点关注：有没有某个 (window, lag) 组合让准确率显著高于随机
3. 如果有 → 说明脑电在特定时间尺度上包含步态阶段信息，可以继续优化
4. 如果没有 → 说明当前数据/采样条件下，脑电确实不适合做步态二分类

这个结论本身就是有价值的——它会直接影响后续脑机接口的算法架构选择。
