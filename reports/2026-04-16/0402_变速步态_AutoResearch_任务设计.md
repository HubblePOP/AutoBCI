# 0402 变速步态 AutoResearch 任务设计

更新时间：2026-04-16

## 目标

回答一个比“支撑 / 摆动”更上游的问题：

> `20240402` 这一天的运动数据里，能不能稳定识别出不同速度段？

这里先不直接做“脑电预测速度”，而是先把运动侧的速度段定义做扎实。  
原因是如果速度标签本身不稳，后面的脑电任务会被错误标签带偏。

## 为什么值得单独做成 AutoResearch

`0402` 已知是变速数据，而且文件名已经带有明显的速度档：

- `walk_12km_01.xlsx`
- `walk_12km_02.xlsx`
- `walk_15km_01.xlsx`
- `walk_15km_02.xlsx`
- `walk_18km_01.xlsx`
- `walk_18km_03.xlsx`
- `walk_18km_04.xlsx`
- `walk_21km_02.xlsx`
- `walk_21km_03.xlsx`
- `walk_24km_01.xlsx`
- `walk_24_30km_01.xlsx`

这意味着它天然适合拿来做一个“速度状态能否从运动学里被自动分出来”的研究问题。

## 这条任务不要直接复用当前步态二分类主线

当前 `program.current.md` 还是在做：

- `gait_phase_eeg_classification`
- 标签是 `支撑 / 摆动`
- 问题是“哪个窗长和固定全局时延最有信息”

如果直接把这一套强行改成速度任务，会把两个问题混在一起：

1. 步态相位能不能切稳
2. 速度状态能不能区分

这会让实验解释变得很差。

更合理的做法是给 `0402` 单开一条任务线。

## 推荐的任务分三层

### 第一层：运动侧速度状态发现

先完全不看脑电，只看运动学。

目标：

- 判断 `12 / 15 / 18 / 21 / 24 / 24-30` 这些速度档，能不能从运动特征里被区分开
- 找到最稳定的速度代理特征

候选特征：

- 单肢步频，也就是 cadence
- 单肢支撑占比，也就是 duty factor
- 同侧前后肢同时摆动占比
- toe 振幅
- `TOE_z` 上升段和下降段持续时间
- `MTP` 与 `TOE` 的相位差
- marker 轨迹的一阶差分速度 `speed_rms`

这一步的目标不是得一个“最强模型”，而是回答：

> 速度段有没有稳定结构，哪些特征最能把它们分开？

### 第二层：变速边界检测

如果某些试次内部不是单一速度，而是中途换挡，就做边界检测。

目标：

- 在一条连续试次里自动找出速度切换点
- 判断速度变化是突变还是缓变

这一步可比较：

- 固定窗口分类再做平滑
- 变化点检测
- 基于步频 / duty factor 的规则分段

### 第三层：脑电能否预测速度状态

只有在前两层把运动侧速度标签做稳之后，才值得上脑电。

目标：

- 看脑电里是否包含和速度状态相关的信息
- 先做粗粒度 `slow / medium / fast`
- 再考虑更细的多档速度

## 我建议的 AutoResearch Program 边界

### Program ID

`gait_speed_0402_variable_speed`

### Program 目标

在 `20240402` 的变速跑步机试次中，判断不同速度档是否能从运动学中被稳定识别，并构造可用于后续脑电解码的速度状态标签。

### 允许的方向

- 只分析 `20240402`
- 允许在 `TOE / MTP / ANK` 等脚部相关 marker 上试不同特征
- 允许做规则法、树模型、序列模型比较
- 允许把速度做粗分箱，例如 `slow / medium / fast`

### 不允许的方向

- 不直接混入 `0717 / 0719`
- 不直接改当前步态二分类主线的标签口径
- 不把速度任务和支撑 / 摆动任务混成一个 benchmark

## 第一版最值得跑的标签定义

### 方案 A：按文件名速度做监督标签

直接把文件名里的速度档当标签：

- `12`
- `15`
- `18`
- `21`
- `24`
- `24_30`

优点：

- 最快
- 能立刻判断“运动学特征到底能不能把这些速度档分出来”

风险：

- 文件名不一定等于实际瞬时速度
- `24_30` 这种混合档可能会污染标签

### 方案 B：先做三档粗标签

把速度先并成：

- `slow = 12 / 15`
- `medium = 18 / 21`
- `fast = 24 / 24_30`

优点：

- 更稳
- 更适合早期 AutoResearch 先回答“有没有信号”

风险：

- 会丢细粒度信息

### 当前推荐

先做 **方案 B**。  
也就是第一轮先回答：

> 运动学里能不能稳定分出 `slow / medium / fast` 三个速度状态？

## 建议的评估指标

### 运动侧分类

- `balanced_accuracy`
- `macro_f1`
- 混淆矩阵

### 速度边界检测

- 边界时间误差
- 分段 purity
- 每段平均速度代理是否单调

### 解释性指标

- 各速度档的步频分布
- 各速度档的支撑占比分布
- `TOE` 与 `MTP` 相位差分布

## 第一轮建议的 tracks

### Track 1：规则特征 + XGBoost

目标：

- 用最容易解释的统计特征先判断三档速度能不能分开

输入特征：

- cadence
- duty factor
- toe 振幅
- speed_rms
- 同侧同时 swing 占比

### Track 2：规则特征 + Ridge

目标：

- 作为最简单的线性对照

### Track 3：窗口序列 + feature_gru

目标：

- 判断速度变化是否具有短时序结构，而不是单窗统计就够了

### Track 4：窗口序列 + feature_tcn

目标：

- 对比 TCN 是否比 GRU 更适合这类变速段识别

## 第一轮不建议跑什么

- 不建议一上来就做脑电到速度
- 不建议一上来就做 6 档精细分类
- 不建议把 `24_30` 和普通单速文件混在同一个干净监督集里不加标记

## 预期输出

第一轮 AutoResearch 最理想的产物不是“冠军分数”，而是这三样：

1. 一张速度档可分性的结论表  
   哪些速度档能分开，哪些混得最厉害

2. 一张解释性特征表  
   哪些特征最能解释速度变化

3. 一版可冻结的粗速度标签  
   供下一轮 EEG 任务使用

## 我建议的执行顺序

1. 先做 `0402` 运动侧三档速度 benchmark
2. 如果三档都分不开，先不要上脑电
3. 如果三档分得开，再做速度边界检测
4. 只有边界也稳了，才把它升级成脑电任务

## 结论

能安排 AutoResearch，而且值得做。  
但**正确的起点不是“脑电直接判速度”，而是“先用运动数据把速度状态任务定义清楚”**。

如果下一步要真的开跑，我建议就按这条顺序：

- `0402` 单日
- 先运动侧
- 先三档速度
- 先解释性强的特征和树模型
- 再考虑序列模型
- 最后才上脑电
